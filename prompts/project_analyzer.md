# 项目记忆分析器

## 任务

从原材料中提取和这个甲方相关的项目事实，构建 Project Memory 知识库。

## 提取维度

### 1. 项目概览
- 项目类型
- 行业与业务背景
- 当前阶段
- 交付对象
- 成功标准

### 2. 核心目标
- 甲方嘴上说的目标
- 真实最优先目标
- 业务 KPI / 品牌目标 / 老板诉求
- 什么比“把活做完”更重要

### 3. 审批链
- 谁表面在提需求
- 谁真正拍板
- 哪些反馈来自“老板说”
- 哪些角色会在最后阶段突然出现

### 4. 风格偏好
- 喜欢的表达方式
- 喜欢的视觉方向
- 经常参考的品牌 / 案例 / benchmark
- 高频关键词（如“高级感”“简洁”“大气”“不要互联网味”）

### 5. 红线与禁区
- 明确说过不要的内容
- 法务 / 品牌 / 老板敏感点
- 预算 / 节点上的硬限制
- 哪种方案一定会被打回

### 6. 修改历史
- 需求如何变化
- 哪类变更最常见
- 哪些看似定稿、其实仍会反复
- 临上线新增需求的模式

### 7. 风险档案
- 常见延误原因
- 最容易返工的环节
- 最危险的错误预判
- 过往踩坑记录

## 输出格式

```markdown
## 项目概览
- 行业/项目：{xxx}
- 当前阶段：{xxx}
- 甲方角色：{xxx}

## 核心目标
- 明面目标：{xxx}
- 真实优先级：{xxx}

## 审批链
- 日常接口人：{xxx}
- 真正拍板人：{xxx}
- 隐性 stakeholder：{xxx}

## 风格偏好
- 喜欢：{xxx}
- 讨厌：{xxx}
- 高频关键词：{xxx}

## 红线与禁区
- {xxx}

## 修改历史
- {按时间或阶段列出}

## 风险档案
- {最容易返工的点}
```

## 注意事项

- 文档里的“官方目标”和聊天里的“真实目标”可能不一致，要同时保留
- 聊天中的随口一句“老板不喜欢”往往比正式 PRD 更关键
- 如果信息不足，标注 `[待补充]`
