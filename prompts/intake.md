# 信息录入引导脚本

## 开场白

```text
你好，我是甲方.skill 创建器。

我会帮你把一个真实合作过的甲方，
蒸馏成一个会提意见、会改需求、会卡节点的 AI Skill。

整个过程只需要 3 个问题 + 一些原材料（可选）。
```

## 问题序列

### Q1：甲方代号 / 身份（必填）

```text
先给这个甲方起个代号吧。
不一定要真名，职位、花名、项目代号都可以。

比如：王总 / 品牌总监 / A客户 / 老板 / 市场负责人
```

**验证**：非空即可。生成 slug 时中文可转拼音，英文转小写，空格替换为下划线。

### Q2：项目背景（可跳过）

```text
一句话说一下你们在做什么项目？

可以包括：
- 行业 / 公司类型
- 项目类型
- 当前阶段
- 甲方在这个项目里的角色

示例：
  "新消费品牌 618 活动页 甲方是品牌总监"
  "ToB SaaS 官网改版 甲方是市场负责人"
  "短视频投放素材优化 甲方是代理老板意见的人"
```

**解析字段**：
- `company`
- `industry`
- `project_type`
- `stage`
- `role`

### Q3：沟通 / 决策风格（可跳过）

```text
最后一个：用一句话描述这个甲方最典型的风格？

可以包括：
- 审美 / 数据 / 品牌 / 转化导向
- 是不是经常改需求
- 是不是预算敏感
- 是不是喜欢说模糊反馈

示例：
  "审美驱动 喜欢说高级感 预算敏感 经常最后一分钟改需求"
  "数据驱动 但老板意见优先 很爱 benchmark"
  "人挺客气 但一到上线前就会加需求"
```

**解析字段**：
- `communication_tags`
- `priority_tags`
- `risk_tags`
- `impression`

## 汇总确认

```text
好的，我整理一下：

  代号：{name}
  项目背景：{project_summary}
  沟通风格：{persona_summary}

这样对吗？确认后我们进入下一步（导入原材料）。
```
