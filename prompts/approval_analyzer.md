# 甲方人格分析器

## 任务

从原材料中提取甲方的说话风格、决策偏好和协作行为，构建可驱动反馈的 Approval Persona。

## 提取维度

### 1. 说话风格
- 高频词和口头禅
- 消息长度
- 语气强度
- 标点和格式习惯
- 是直接否定还是绕着说

### 2. 反馈方式
- 喜欢给明确标准还是给感觉
- 先说优点还是直接打回
- 常用模糊词（“再高级一点”“先顺一顺”“还是不对”）
- 喜欢问问题还是直接下指令

### 3. 决策顺序
- 品牌感
- 转化
- 老板满意
- 预算
- 时间节点
- 风险控制

需要判断这些因素真实的优先级，而不是表面顺序。

### 4. 变更行为
- 什么时候最爱改需求
- 变更原因通常是什么
- 是否经常临上线追加内容
- 是否倾向“既要又要”

### 5. 协作行为
- 回复快慢
- 是否看文档
- 是否拉人进群
- 是否推责任
- 是否口头说过、群里没留痕

## 标签翻译表

| 用户标签 | 翻译为行为规则 |
|---------|----------------|
| 审美驱动 | 先看气质和感觉，能接受解释不清，但无法接受“不高级” |
| 数据驱动 | 习惯问结果和指标，没有数据支持就不放心 |
| 品牌导向 | 更重视品牌统一性和长期印象，不愿意为了短期转化牺牲调性 |
| 转化导向 | 一切围绕点击、留资、成交，愿意接受更直接的表达 |
| 预算敏感 | 先问成本，再问效果，经常要求“看起来贵，但做起来别太贵” |
| 时间焦虑 | 对节点极度敏感，宁可简化方案也不愿延期 |
| 喜欢 benchmark | 说不清楚时会甩参考图或竞品，习惯用“像这个”表达需求 |
| 老板意志代理人 | 自己不一定有明确偏好，但会频繁转述更高层意见 |
| 模糊型甲方 | 经常使用感觉词，不愿意一次性给清楚标准 |
| 既要又要 | 很难接受取舍，希望保留所有好处且没有代价 |
| 不看文档 | 主要靠口头和截图交流，对长文耐心低 |
| 临上线改需求 | 关键节点前会突然新增内容，且认为这是正常优化 |

## 输出格式

输出一个 5 层 Persona 结构草稿（详见 approval_builder.md），每层包含具体行为规则，而不是抽象标签。
