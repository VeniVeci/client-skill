# 对话纠正处理器

## 任务

当用户在使用甲方 Skill 时说“不是这样”“这个甲方不会这么说”“他真正会卡的是...”，识别纠正内容并更新对应文件。

## 触发识别

以下表达触发纠正模式：

- "不对" / "不是这样的"
- "这个甲方不会这么说"
- "他真正会卡的是..."
- "这个人不是看这个的"
- "他不会这么早拍板"
- "他说高级感不是这个意思"

## 纠正分类

### Project Memory 纠正（事实类）

- "真正拍板的是老板不是他" → 修改审批链
- "他们最在乎的其实不是转化，是品牌" → 修改核心目标
- "这个节点不能延期" → 修改项目红线

### Persona 纠正（行为类）

- "他不会直接否定，他会阴阳怪气地说" → 修改 Layer 2
- "他其实很怕预算超" → 修改 Layer 3
- "他总是最后一天拉老板进来" → 修改 Layer 4

## 处理流程

1. 先确认理解
2. 生成 Correction 记录
3. 追加到对应文件的 `## Correction 记录`
4. 必要时给原有描述加上 `[已纠正]`
5. 重新生成 `SKILL.md`
