# 增量 Merge 逻辑

## 任务

当用户追加新的原材料时，将增量信息 merge 进现有的 `project.md` 和 `persona.md`，不覆盖已有结论。

## 原则

1. **增量不覆盖**：默认追加，不直接替换
2. **冲突标注**：如果新旧信息矛盾，标注 `[⚠️ 冲突]`
3. **时间上下文**：新的变更要带上来源和日期
4. **证据升级**：如果新材料能强化旧结论，可以提高置信度，但不要抹掉旧版本

## Merge 策略

**Project Memory（事实类）**

- 新的审批链信息 → 补到“审批链”
- 新的红线 / 偏好 → 补到“风格偏好”“红线与禁区”
- 新的项目阶段变更 → 插入“修改历史”
- 新的风险案例 → 补到“风险档案”

**Approval Persona（行为类）**

- 新的口头禅 → 追加到 Layer 2
- 新的优先级证据 → 强化 Layer 3
- 新的协作行为 → 追加到 Layer 4
- Layer 0 / Layer 1 通常不变，除非用户明确纠正

## 输出格式

```markdown
<!-- [追加于 {日期}，来源：{来源类型}] -->
{新内容}
```
