# 甲方.skill — 产品需求文档（PRD）

## 产品定位

甲方.skill 是一个运行在 Claude Code 上的 meta-skill。  
用户通过对话式交互导入甲方相关原材料，自动生成一个可独立运行的 Client Persona Skill，用于方案预演、需求对齐、内部培训与项目复盘。

它不是一个“自动接单器”，而是一个“把隐形甲方规则显性化”的工具。

## 核心概念

### 两层架构

| 层 | 名称 | 职责 |
|----|------|------|
| Part A | Project Memory | 存储项目事实：业务目标、审批链、修改历史、禁区、偏好、风险档案 |
| Part B | Approval Persona | 驱动反馈行为：说话风格、决策逻辑、协作方式、常见否定路径 |

两部分既可独立查看，也可组合运行。

### 运行逻辑

```text
用户输入方案 / 提问
  ↓
Part B（Approval Persona）判断：
  甲方会先看什么？会卡什么？会怎么表达？
  ↓
Part A（Project Memory）补充：
  当前项目背景、历史修改、老板偏好、红线和预算
  ↓
输出：
  用甲方的方式给出反馈
```

### 进化机制

```text
追加新材料 → 增量分析 → merge 进现有 Skill
对话纠正 → 识别修正点 → 写入 Correction 层
版本管理 → 每次更新自动存档 → 支持回滚
```

## 用户旅程

```text
用户触发 /create-client
  ↓
[Step 1] 基础信息录入（3个问题）
  - 甲方代号 / 角色
  - 项目背景
  - 沟通风格 / 决策特征
  ↓
[Step 2] 原材料导入
  - 聊天记录
  - 邮件 / 评论 / 批注
  - PRD / 提案 / 方案
  - 会议纪要 / 转写
  - 直接粘贴经验总结
  ↓
[Step 3] 自动分析
  - 线路 A：提取 Project Memory
  - 线路 B：提取 Approval Persona
  ↓
[Step 4] 生成预览，用户确认
  - 项目记忆摘要
  - 甲方人格摘要
  ↓
[Step 5] 写入文件，立即可用
  - 生成 clients/{slug}/ 目录
  - 包含 SKILL.md（完整组合版）
  - 包含 project.md 和 persona.md（独立部分）
  ↓
[持续] 进化模式
  - 追加新材料
  - 对话纠正
  - 自动版本存档
```

## 安全边界

1. **仅用于内部协作、训练、复盘和预演**
2. **不冒充真人，不替代正式审批**
3. **数据默认本地存储**
4. **涉及客户机密时要求用户自行脱敏**
5. **Layer 0 硬规则**保证不生成明显超出证据范围的审批结论

## 数据源支持矩阵

| 来源 | 格式 | 提取内容 | 优先级 |
|------|------|---------|--------|
| 聊天记录 | txt / json / md | 语气、回复时机、否定方式、压力表达 | ⭐⭐⭐ |
| PRD / brief / proposal | md / txt / pdf | 业务目标、交付标准、禁区 | ⭐⭐⭐ |
| 会议纪要 / 转写 | md / txt | 隐性拍板人、真实优先级、变更来源 | ⭐⭐⭐ |
| 评论 / 批注 | txt / json | 审美偏好、反馈模板、常见反对理由 | ⭐⭐ |
| 邮件 | txt / eml 转文本 | 正式反馈、审批轨迹、风险措辞 | ⭐⭐ |
| 手动口述 | 纯文本 | 团队经验、踩坑总结、潜规则 | ⭐ |

## 文件结构

```text
clients/
  └── {slug}/
      ├── SKILL.md          # 完整组合版，可直接运行
      │                     # 触发词: /{slug}
      ├── project.md        # Part A：项目记忆
      │                     # 触发词: /{slug}-project
      ├── persona.md        # Part B：甲方人格
      │                     # 触发词: /{slug}-persona
      ├── meta.json         # 元信息
      ├── versions/         # 历史版本存档
      └── sources/          # 原始材料存放
          ├── chats/
          ├── briefs/
          ├── meetings/
          └── comments/
```

## 爆点设计

这个项目想要成立，不只是“能跑”，还要“一眼能懂、一句话能传”。

核心传播句式：

```text
把你的甲方蒸馏成 AI Skill，
让它在你提案之前先把你打回去。
```

为什么它比泛泛的 “AI PM / AI Reviewer” 更容易传播：

1. 所有人都见过甲方，而且都有怨气和段子
2. 截图演示天然有戏剧冲突
3. 公开和半公开的项目材料很常见，容易做样例
4. 同时具备娱乐性和工作价值

## 与同事.skill / 前任.skill 的关系

| 维度 | 同事.skill | 前任.skill | 甲方.skill |
|------|-----------|-----------|-----------|
| Part A | Work Skill | Relationship Memory | Project Memory |
| Part B | Persona（工作场景） | Persona（关系场景） | Approval Persona（审批场景） |
| 核心价值 | 复刻同事经验 | 复刻人物记忆 | 复刻甲方反馈逻辑 |
| 主要数据 | 工作文档/消息 | 聊天/照片/回忆 | brief/批注/会议/聊天 |
| 主要场景 | 写代码/问问题 | 聊天/回忆 | 改方案/预演/复盘 |
| 传播钩子 | 离职同事赛博永生 | 情感回忆杀 | 甲方又改需求了 |
